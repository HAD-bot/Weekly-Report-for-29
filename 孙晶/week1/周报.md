# git

## 版本控制器的方式

a.集中式版本控制工具

版本库是集中存放在中央服务器的。个人工作时要从中央服务器下载代码，修改后提交到中央服务器，需要联网才能工作。SVN CVS

b.分布式版本控制工具

每个人的电脑都是一个完整的版本库，无需联网，多人协作是只需将自己的修改推送给对方（一般不用）,一般还是有共享版本库。GIT

## 为常用指令配置别名

在用户下创建配置文件

```bash
touch ~/.bashrc
```

在文件中配置别名

```bash
#用于输出git提交日志
alias git-log='git log --pretty=online --all --graph --abbrev-commit'
#用于输出当前目录所用文件及基本信息
alias ll='ls -al'
```

执行文件

```bash
source ~/.bashrc
```



## 获取本地仓库

打开任意一个空目录，进入，右键打开git bash窗口，执行命令，创建成功后可以在文件夹下看到.git目录。

```bash
git init
```

### 基本操作指令

![f033e1ee4cd2d36248e61d155e3994b3](C:\Users\PC\Documents\Tencent Files\3631076978\nt_qq\nt_data\Pic\2026-07\Ori\f033e1ee4cd2d36248e61d155e3994b3.jpg)

创建文件，在这个目录下用touch

```bash
touch file01.txt
```

查看文件状态

```bash
git status
```

由工作区全部加入暂存区

```bash
git add .（或者文件名）
```

由暂存区到仓库

```bash
git commit （或者文件名） -m "注释"
```

查看提交记录

```bash
git log #后面可以加选项
--all #显示所有分支
--pretty==online #将提交信息显示为一行
--abbrev-commit #使得输出的commitld更简短
-graph #以图的形式显示
```

编辑文档，编辑之后查看状态变成工作区

```bash
vi file01.txt    #进入普通模式，想要输入按i，退出esc，然后输入冒号，退出并保存：wq，退出不保存：q！，保存：w
```

清除命令

```bash
clear
```

版本退回

退回之前的ID，用 git log，找到commitID；退回已删除的,用git reflog,找commitID

```bash
git reset --hard commitID
```

忽略文件

创建忽略文件,在忽略文件中的文件，不会被提交（add.),**不会纳入git管理**，查看状态也不会显示

```bash
touch .gitignore
```

mv命令

1. 移动文件 / 文件夹（剪切）

语法：`mv 源文件路径 目标文件夹`

```bash
# 把 a.txt 移动到 test 文件夹
mv a.txt ./test/
# 把整个文件夹 dir1 移到桌面
mv dir1 ~/Desktop/
```

2. 重命名文件 / 文件夹

语法：`mv 旧文件名 新文件名`

```bash
# 将 old.txt 改名为 new.txt
mv old.txt new.txt
# 文件夹改名
mv git.test git_demo
```

### 分支

查看本地分支

~~~bash
git branch
~~~

创建分支，创建的分支和当前分支内容一致

~~~bash
git branch 分支名
~~~

切换分支，HEAD指向谁，谁就是当前分支

~~~bash
git checkout 分支名
~~~

创建并且切换到这个分支

~~~bash
git checkout -b 分支名
~~~

把其他分支合并到master

~~~bash
git merge dev01
~~~

删除分支，需要做各种检查(没合并)

~~~bash
git branch -d 分支名
~~~

强制删除

~~~bash
git branch -D 分支名
~~~

解决冲突，手动解决，然后提交

### **配置SSH公钥**

生成SSH公钥,如果公钥已存在自动覆盖

~~~bash
ssh-keygen -t rsa
~~~

gitee设置账户公钥

获取公钥用cat，配置到gitee，然后验证

~~~bash
ssh -T git@gitee.com
~~~

### **操作远程仓库**

把本地仓库链接创建的远程仓库

~~~bash
git remote add origin git@gitee.com:had-bot/git_test.git
git remote add <远端名> <仓库路径>
#远端名默认是origin，从远端服务器获取URL
~~~

![image-20260708171427542](C:\Users\PC\AppData\Roaming\Typora\typora-user-images\image-20260708171427542.png)

查看本地连接的仓库

~~~bash
git remote
~~~

将本地仓库的文件推到远程仓库

~~~bash
git push origin master#名字一样省略
git push [-f] [--set-upstream] [远端名称[本地分支名][:远端分支名]]
#-f强制覆盖
#--set-upstream推送远端的同时并且建立起和远端分支的关系联系
git push --set-upstream origin master:master
#有了对应关系可以直接git push
~~~

看本地分支和远程分支的对应关系

~~~bash
git branch -vv
~~~

### 抓取和拉取

抓取 将远程仓库更新拉到本地，不会合并,需要在执行git merge

~~~bash
git fetch [remote name] [branch name] #如果不指定就是所有
~~~

拉取  将远程仓库更新拉到本地并合并

~~~bash
git pull [remote name] [branch name] #如果不指定就是所有
~~~

### 解决冲突

在push之前必须先pull，有冲突，手动操作文件，改完之后，重新提交
