# git

## 版本控制器的方式

### a.集中式版本控制工具

版本库是集中存放在中央服务器的。个人工作时要从中央服务器下载代码，修改后提交到中央服务器，需要联网才能工作。SVN CVS

### b.分布式版本控制工具

每个人的电脑都是一个完整的版本库，无需联网，多人协作是只需将自己的修改推送给对方（一般不用）,一般还是有共享版本库。GIT

## 为常用指令配置别名

在用户下创建配置文件

```bash
touch ~/.bashrc
```

在文件中配置别名

```bash
#用于输出git提交日志
alias git-log='git log --pretty=online --all --graph --abbrev-commit'
#用于输出当前目录所用文件及基本信息
alias ll='ls -al'
```

执行文件

```bash
source ~/.bashrc
```



## 获取本地仓库

打开任意一个空目录，进入，右键打开git bash窗口，执行命令，创建成功后可以在文件夹下看到.git目录。

```bash
git init
```

## 基本操作指令

![f033e1ee4cd2d36248e61d155e3994b3](C:\Users\PC\Documents\Tencent Files\3631076978\nt_qq\nt_data\Pic\2026-07\Ori\f033e1ee4cd2d36248e61d155e3994b3.jpg)

创建文件，在这个目录下用touch

```bash
touch file01.txt
```

查看文件状态

```bash
git status
```

由工作区全部加入暂存区

```bash
git add .（或者文件名）
```

由暂存区到仓库

```bash
git commit （或者文件名） -m "注释"
```

查看提交记录

```bash
git log #后面可以加选项
--all #显示所有分支
--pretty==online #将提交信息显示为一行
--abbrev-commit #使得输出的commitld更简短
-graph #以图的形式显示
```

编辑文档，编辑之后查看状态变成工作区

```bash
vi file01.txt    #进入普通模式，想要输入按i，退出esc，然后输入冒号，退出并保存：wq，退出不保存：q！，保存：w
```

清除命令

```bash
clear
```

##  版本退回

退回之前的ID，用 git log，找到commitID；退回已删除的,用git reflog,找commitID

```bash
git reset --hard commitID
```
